# S3 Bucket Security Configuration

## Overview

This document describes the security configurations applied to S3 buckets in the Green Passport platform to ensure data protection, access control, and compliance with security best practices.

## Security Requirements

Based on Requirements 22.1, 22.3, and 22.4:

1. **Block all public access** on QR codes bucket
2. **Configure bucket policy** for signed URL access
3. **Enable encryption at rest** (AES-256)
4. **Enable versioning** on critical buckets

## Bucket Security Configurations

### 1. QR Codes Bucket (`gp-qr-codes-{environment}`)

**Purpose**: Store QR code images with private access

**Security Features**:
- ✓ Block all public access (BlockPublicAcls, IgnorePublicAcls, BlockPublicPolicy, RestrictPublicBuckets)
- ✓ AES-256 encryption at rest
- ✓ Versioning enabled
- ✓ Bucket policy for signed URL access only
- ✓ HTTPS-only access (deny insecure transport)

**Access Method**: Signed URLs with 1-hour expiration generated by Lambda functions

**Bucket Policy**:
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "DenyInsecureTransport",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:*",
      "Resource": [
        "arn:aws:s3:::gp-qr-codes-{environment}",
        "arn:aws:s3:::gp-qr-codes-{environment}/*"
      ],
      "Condition": {
        "Bool": {
          "aws:SecureTransport": "false"
        }
      }
    },
    {
      "Sid": "AllowSignedURLAccess",
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::{account-id}:root"
      },
      "Action": [
        "s3:GetObject",
        "s3:PutObject"
      ],
      "Resource": "arn:aws:s3:::gp-qr-codes-{environment}/*"
    }
  ]
}
```

### 2. Frontend Assets Bucket (`gp-frontend-{environment}`)

**Purpose**: Host static frontend assets served via CloudFront

**Security Features**:
- ✓ AES-256 encryption at rest
- ✓ CORS configuration for API access
- ✓ Static website hosting enabled
- ✓ CloudFront Origin Access Identity (OAI) for controlled access

**Access Method**: CloudFront distribution with OAI

**Note**: Public access is not blocked on this bucket as it serves static website content through CloudFront. Access is controlled via CloudFront OAI.

## Signed URL Generation

Lambda functions generate signed URLs for temporary access to QR code downloads:

```typescript
import { S3Client, GetObjectCommand } from '@aws-sdk/client-s3';
import { getSignedUrl } from '@aws-sdk/s3-request-presigner';

const client = new S3Client({ region: 'us-east-1' });

async function generateSignedUrl(bucketName: string, key: string): Promise<string> {
  const command = new GetObjectCommand({
    Bucket: bucketName,
    Key: key
  });

  // Generate signed URL with 1-hour expiration
  const signedUrl = await getSignedUrl(client, command, {
    expiresIn: 3600 // 1 hour
  });

  return signedUrl;
}
```

## Security Best Practices

1. **Never expose bucket names publicly**: Use environment variables
2. **Rotate access keys regularly**: Use IAM roles for Lambda functions
3. **Monitor access logs**: Enable S3 access logging for audit trails
4. **Use least privilege**: Grant only necessary permissions to each Lambda function
5. **Encrypt in transit**: Enforce HTTPS-only access via bucket policy
6. **Version critical data**: Enable versioning to protect against accidental deletion
7. **Regular security audits**: Periodically verify security configurations

## Compliance

These configurations meet the following security standards:

- **Requirement 22.1**: All public access blocked on QR codes bucket
- **Requirement 22.3**: AES-256 encryption at rest enabled
- **Requirement 22.4**: Versioning enabled on critical buckets
- **Requirement 10.3**: Signed URLs with 1-hour expiration for temporary access
- **Requirement 10.5**: Access denied when signed URL expires

## Troubleshooting

### Issue: Access Denied when accessing QR codes

**Solution**: Ensure you're using a valid signed URL generated by the Lambda function. Direct access is blocked.

### Issue: Bucket policy conflicts with public access block

**Solution**: The QR codes bucket has public access blocked. Access is only allowed through signed URLs generated by authorized Lambda functions.

### Issue: Frontend assets not loading

**Solution**: Verify CloudFront distribution is configured with correct Origin Access Identity (OAI) to access the frontend bucket.

## Monitoring

Monitor S3 security with CloudWatch metrics:

- `NumberOfObjects`: Track object count
- `BucketSizeBytes`: Monitor storage usage
- `AllRequests`: Track access patterns
- `4xxErrors`: Monitor access denied errors
- `5xxErrors`: Monitor server errors

Set up CloudWatch alarms for:
- Unusual access patterns
- High error rates
- Unauthorized access attempts
